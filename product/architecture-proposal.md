## 一、核心设计原则（先对齐这个）

| 原则 | 现状 | 建议 |
| --- | --- | --- |
| **架构视角** | 功能优先（新生入学 / 课程学习 / …） | **切换为"主题优先"**，贴近源文档"你好新同学 / 学在湘大 / …"的用户心智 |
| **一内容多归属** | 每篇 md 只能 `category: X`，出现在一个目录 | 引入 `sections: [A, B]` 数组，让"湘大地图 / 奖学金政策"这类跨主题内容能在多个入口展示 |
| **Q&A 独立层** | 4 篇 Q&A 打散在 01/02/06 | **抽出成 `/qa` 顶级入口**，源文档就是这样处理的 |
| **反诈骗 / 公告栏** | 完全缺失 | 首页顶部承载 + 独立 `/about` 页承载详细内容 |
| **子文档跳转** | 依赖 `knownSubcardTitles` 硬编码白名单 | 移除白名单，改为**基于目录 / tags 自动生成"该主题下相关资料"** |
| **guide 页 4 模块** | 硬编码指向 3 篇"总览"文件 | 改为**按新生入学时间线**（报到前 → 报到 → 军训 → 开学后）自动串起来 |

│

│
├── ⭐ 相关内容板块（新逻辑）
│   ├── 同一 section 下的其他资料（最多 6 篇）
│   ├── 共同 tags 命中最多的 3 篇

- 面包屑帮用户理解自己在哪个主题下。
- **移除 subcard 硬编码逻辑**。

## 九、数据模型建议（frontmatter 升级）

```yaml

# ⭐ 新字段

sections: [study, newcomer]        # 也出现在的其他栏目卡片列表
phase: [first-week, arrival]       # 时间线归属（可多个）
tags: [选课, 培养方案, 学分, 教务]    # 用于相关推荐
weight: 5                          # section 内排序（越大越靠前）
```

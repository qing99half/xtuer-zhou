# 资料入库与审核规则

本项目资料遵循“先待整理、再审核、后入库”的流程。

## 资料状态

### 待整理

待整理资料存放在 `docs/99-待整理资料/`。这些资料可以用于产品分析、内容拆分和开发排期，但不能直接进入正式资料库页面，也不能进入 RAG 索引。

### 已审核

已审核资料是经过用户确认后，可以进入对应正式目录的资料。只有 `status: 已审核` 的 Markdown 文件才允许在资料库页面展示。

## 入库流程

1. 接收用户提供资料。
2. 保存到 `docs/99-待整理资料/`。
3. 提取文本并标注来源。
4. 用户确认资料可入库范围。
5. 拆分为正式 Markdown 文档。
6. 补充 frontmatter。
7. 进入资料库展示，并为后续 RAG 索引准备。

## 红线

- 不直接使用未经确认的网络信息作为正式资料。
- 不把待整理资料加入 RAG 索引。
- AI 生成内容不能直接作为事实资料入库。
- 对资料不足的问题，系统应提示资料库暂未覆盖，不能编造答案。
